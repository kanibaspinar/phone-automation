API Documentation

All endpoints are prefixed with /api

Base URL & Response Format

The server runs on http://localhost:5000 by default. All routes below are relative to /api.

Standard success response:

{
  "success": true,
  "data": { ... }        // varies per endpoint
}

Standard error response:

{
  "error": "human-readable error message"
}

Task-queued response (most action endpoints):

{
  "success": true,
  "message": "Task description",
  "task_id": "uuid-v4"
}

HTTP Status Codes

CodeMeaning
200OK — request succeeded
201Created — resource successfully created
400Bad Request — missing or invalid fields
404Not Found — device / account / task does not exist
409Conflict — resource already exists
500Server Error — unexpected exception
503Service Unavailable — task manager not initialised

Instagram — Account Creation

Automates new Instagram account registration on a connected Android device using UI automation (uiautomator2) and automatic email OTP verification via IMAP. All jobs run in the background — poll /creation-status/{task_id} for progress.

POST /instagram/accounts/create-new

Create New Instagram Account

Launches a background job that opens Instagram on the target device and completes the full signup flow (contact → OTP → birthday → name → password → username). Returns 202 Accepted immediately with a task_id.

FieldTypeRequiredDescription
device_idstringrequiredADB serial of the target device (e.g. 38R9X18919G04940)
usernamestringrequiredDesired Instagram username. A numeric suffix is appended automatically if already taken.
passwordstringrequiredAccount password (min 6 characters)
emailstringoptional*Registration email address. Required if phone_number is omitted.
email_passwordstringoptional*Email account password for IMAP OTP retrieval. Required when email is provided.
phone_numberstringoptional*Phone number including country code (e.g. +14155550100). Required if email is omitted.
full_namestringoptionalDisplay name. Defaults to a title-cased version of username.
birthdaystringoptionalISO date YYYY-MM-DD. Defaults to 1995-06-15.
imap_serverstringoptionalOverride IMAP hostname. Auto-detected from email domain if omitted (Gmail, Outlook, Yahoo, etc.).
imap_portintegeroptionalIMAP port. Defaults to 993 (SSL).

* Either email + email_password or phone_number must be supplied.

Request example (email flow):

POST /api/instagram/accounts/create-new
Content-Type: application/json

{
  "device_id":       "38R9X18919G04940",
  "username":        "mynewaccount",
  "password":        "Str0ngP@ss!",
  "email":           "myemail@gmail.com",
  "email_password":  "gmail_app_password",
  "full_name":       "My Name",
  "birthday":        "1998-03-22"
}

Response (202):

{
  "success": true,
  "message": "Account creation started",
  "task_id": "3f2d1a8b-4c56-7e89-abcd-ef0123456789"
}

POST /tiktok/actions/run-collection

Run Mass Collection

Starts the full collection loop: fetches followers of targets via the evelode.com follower API (cursor-paginated), filters by quality criteria, then performs a randomised mix of follow / like / comment / visit / story-like actions up to the daily limits. Uses smart delay (time-window ÷ total actions × weight ± 20%) between each action. The loop runs until all daily limits are hit, stop_time is reached, or a stop signal is sent via POST /tiktok/tasks/{task_id}/stop.

Config is loaded from the account's saved settings. Any field sent in the request body overrides the saved value for this run.

FieldTypeRequiredDescription
device_idstringrequired
usernamestringrequiredMust exist in DB with targets configured
targetsstringoverrideComma/newline competitor usernames
start_timestringoverrideHH:MM
stop_timestringoverrideHH:MM
daily_follow_limitintegeroverride
daily_like_limitintegeroverride
daily_comment_limitintegeroverride
daily_visit_limitintegeroverride
daily_story_like_limitintegeroverride
unfollow_limitintegeroverride
min_followers / max_followersintegeroverrideFollower count quality filter
min_following / max_followingintegeroverrideFollowing count quality filter
min_posts / max_postsintegeroverridePost count quality filter
language_codestringoverridee.g. en,tr
genderstringoverridemale | female | both
comment_textsstringoverrideNewline-separated comment pool

{
  "device_id": "emulator-5554",

  "gender": "female",
  "language_code": "en"
}

Manage proxy6.net proxies. The proxy chain is: Device → Gnirehtet (USB) → PC local forwarder → proxy6.net → Internet. Credentials are injected by the local forwarder; the device never sees them.

Get Proxy Config

Returns masked API key status and live balance from proxy6.net.

{
  "api_key_set": true,
  "api_key_masked": "abcd…wxyz",
  "balance": { "balance": "12.50", "currency": "USD", "count": 1000 }
}

POST /proxy/assign

Assign Proxy to Device

Starts Gnirehtet for the device, acquires an available proxy from the pool, starts a local TCP forwarder that injects Proxy-Authorization, and sets the Android http_proxy to 10.0.2.2:FORWARDER_PORT. Idempotent — returns existing assignment if already assigned.

FieldTypeDescription
device_idstringREQADB serial of the target device
 
// Fresh assignment:
{
  "ok": true,
  "proxy_id": "12345",
  "host": "45.130.0.1",
  "port": 12345,
  "forwarder_port": 54321   // local PC port the device routes through
}

// Already assigned:
{
  "ok": true,
  "already_assigned": true,
  "proxy_id": "12345",
  "host": "45.130.0.1",
  "port": 12345
}

// No proxies available:
{ "error": "No available proxies in pool" }   // 503
POST /proxy/unassign

Unassign Proxy

Stops the local forwarder, clears the device's http_proxy via ADB, and returns the proxy to the available pool. Pass either proxy_id or device_id.

FieldTypeDescription
proxy_idstringOPTproxy6.net proxy ID to release
device_idstringOPTADB serial — releases the proxy assigned to this device

At least one of proxy_id / device_id is required.

{ "ok": true, "proxy_id": "12345" }

// Nothing was assigned:
{ "ok": true, "note": "No active assignment found" }
POST /proxy/bulk-unassign

Bulk Unassign Proxies

Release multiple proxies at once. Same effect as calling /proxy/unassign for each — stops forwarders, clears device proxies, returns all to pool.

FieldTypeDescription
proxy_idsstring[]REQArray of proxy6.net proxy IDs to release
 
// Request:
{ "proxy_ids": ["12345", "12346", "12347"] }

// Response:
{ "ok": true, "released": 3 }
