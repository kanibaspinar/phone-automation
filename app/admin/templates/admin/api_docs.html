API Documentation

All endpoints are prefixed with /api

Base URL & Response Format

The server runs on http://localhost:5000 by default. All routes below are relative to /api.

Standard success response:

{
  "success": true,
  "data": { ... }        // varies per endpoint
}

Standard error response:

{
  "error": "human-readable error message"
}

Task-queued response (most action endpoints):

{
  "success": true,
  "message": "Task description",
  "task_id": "uuid-v4"
}

HTTP Status Codes

CodeMeaning
200OK — request succeeded
201Created — resource successfully created
400Bad Request — missing or invalid fields
404Not Found — device / account / task does not exist
409Conflict — resource already exists
500Server Error — unexpected exception
503Service Unavailable — task manager not initialised

Devices

GET /devices/list

List Devices

Returns all registered devices. Filter by platform to get devices for a specific use case.

Query Parameters

FieldTypeRequiredDescription

{

  "devices": [
    {
      "id": 1,
      "device_id": "emulator-5554",
      "device_name": "Phone 1",
      "status": "connected",
      "platform": "instagram",
      "assigned_to": null,
      "last_seen": "2026-04-07T10:00:00"
    }
  ]
}

Register a new Android device in the system.

Request Body

Delete Device

Permanently deletes the device and all associated Instagram accounts.

// Response 200

POST /devices/assign

Assign Device to User

Picks N free devices and assigns them to a user ID.

FieldTypeRequiredDescription
user_idstringrequiredIdentifier for the user
countintegeroptionalNumber of devices to assign (default: 1)
POST /devices/{device_id}/unassign

Unassign Device

Removes the user assignment and sets all linked Instagram accounts to logged-out.

GET /devices/user/{user_id}

Get User's Devices

All devices currently assigned to a given user, with their Instagram account count.

GET /devices/free

Free Devices

All devices with no user assignment, plus a summary (total, connected, needs_cleanup).

POST /devices/operations/bulk

Bulk ADB Operations

Run an ADB operation on one or more devices simultaneously.

FieldTypeRequiredDescription
device_idsstring[]requiredList of device serials
operationstringrequiredSee operations table below

Available Operations

ValueAction
clear_instagrampm clear com.instagram.android
clear_tiktokpm clear com.zhiliaoapp.musically
rebootReboot the device
clean_appsDisable common bloatware
install_uiautomatorPush uiautomator2 APKs
start_gnirehtetStart reverse-tethering via gnirehtet
stop_gnirehtetStop gnirehtet
 
// POST /api/devices/operations/bulk
{
  "device_ids": ["emulator-5554", "emulator-5556"],
  "operation": "reboot"
}
// Response
{ "success": true, "results": [
    { "device_id": "emulator-5554", "success": true, "message": "Device reboot initiated" },
    { "device_id": "emulator-5556", "success": true, "message": "Device reboot initiated" }
] }

Bulk Add Accounts

Add multiple accounts in one request. Each object supports the same fields as the single-add endpoint.

POST /tiktok/actions/run-collection

Run Mass Collection

Starts the full collection loop: fetches followers of targets via the evelode.com follower API (cursor-paginated), filters by quality criteria, then performs a randomised mix of follow / like / comment / visit / story-like actions up to the daily limits. Uses smart delay (time-window ÷ total actions × weight ± 20%) between each action. The loop runs until all daily limits are hit, stop_time is reached, or a stop signal is sent via POST /tiktok/tasks/{task_id}/stop.

Config is loaded from the account's saved settings. Any field sent in the request body overrides the saved value for this run.

FieldTypeRequiredDescription
device_idstringrequired
usernamestringrequiredMust exist in DB with targets configured
targetsstringoverrideComma/newline competitor usernames
start_timestringoverrideHH:MM
stop_timestringoverrideHH:MM
daily_follow_limitintegeroverride
daily_like_limitintegeroverride
daily_comment_limitintegeroverride
daily_visit_limitintegeroverride
daily_story_like_limitintegeroverride
unfollow_limitintegeroverride
min_followers / max_followersintegeroverrideFollower count quality filter
min_following / max_followingintegeroverrideFollowing count quality filter
min_posts / max_postsintegeroverridePost count quality filter
language_codestringoverridee.g. en,tr
genderstringoverridemale | female | both
comment_textsstringoverrideNewline-separated comment pool
 
// POST /api/tiktok/actions/run-collection
{
  "device_id": "emulator-5554",
  "username": "myaccount",
  "targets": "competitor1,competitor2",
  "daily_follow_limit": 100,
  "gender": "female",
  "language_code": "en"
}
// Response 200
{
  "success": true,
  "message": "Collection started for myaccount",
  "task_id": "uuid-v4",
  "config": { ... }
}

TikTok — Tasks

GET /tiktok/tasks

All TikTok Tasks

All in-memory TikTok tasks sorted newest-first. Same schema as Instagram tasks.

GET /tiktok/tasks/{task_id}

Task Status

Returns 404 if not found.

POST /tiktok/tasks/{task_id}/stop

Stop Collection Task

Sends a stop signal to a running run_collection task. The loop exits after the current action finishes. Only works for tasks with status running.

// Response 200
{ "success": true, "message": "Stop signal sent to task abc-123" }

// Response 404 (not found or not running)
{ "error": "Task not found or not running" }
